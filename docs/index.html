Motivation

During my undergraduate network engineering coursework, I understood TCP/IP theoretically but had never written code that actually opened a packet. This project was built to close that gap: parse every byte of a live or recorded network stream, reconstruct the protocol stack layer by layer, and surface the results as human-readable statistics.

The secondary goal was engineering discipline — making something that runs correctly on any machine, degrades gracefully when privileges are missing, and can be reproduced from scratch in a few minutes by someone who has never seen the code before.

Technical Highlights

🔬

Multi-layer Protocol Dissection

Parses raw bytes from Ethernet frame through IPv4 header down to TCP/UDP/ICMP payload; detects DNS query names and HTTP method verbs.

⚖️

Dual-mode Architecture

Offline .pcap analysis and live interface sniffing share a single downstream pipeline; live mode falls back silently to a synthetic demo dataset when privileges are insufficient.

📊

Flow-level Statistics

Aggregates per-packet data into time-series metrics: packets per second, peak traffic timestamp, protocol distribution ratios, and top-N source/destination rankings.

🖥️

Rich CLI Dashboard

Real-time terminal output using rich renders colored tables, summary panels, and multi-column layouts with no GUI dependency.

📁

Structured Export

A single --export flag writes a CSV dataset, two PNG charts, and a Markdown summary to ./output/.

🧪

Typed & Tested

Packets are parsed into a typed dataclass; 18 pytest cases cover normal operation, edge cases, and empty input across the analyzer module.

Running the Demo

Fully reproducible locally — one command, no root access, no external datasets.
The system generates its own synthetic demo.pcap (200 packets) at runtime, runs the complete analysis pipeline, and writes all output to ./output/.

Linux / macOS

# Clone and enter the repository
git clone https://github.com/TeWei02/Real-time-Packet-Monitoring-and-Traffic-Statistics-System.git
cd Real-time-Packet-Monitoring-and-Traffic-Statistics-System

# One-command demo runner (creates venv, installs deps, runs demo)
bash run_demo.sh

Windows

git clone https://github.com/TeWei02/Real-time-Packet-Monitoring-and-Traffic-Statistics-System.git
cd Real-time-Packet-Monitoring-and-Traffic-Statistics-System
run_demo.bat

Manual (Python 3.11+ already installed)

pip install -r requirements.txt
python main.py --demo --export

Both scripts detect the Python interpreter automatically, create an isolated venv if one does not exist, and exit with a clear error message if any step fails.

Learning Outcomes

Building this project required engaging with problems that are not well-covered in introductory coursework. Specifically, it deepened my understanding of:

  • How the TCP/IP protocol stack maps to actual byte offsets in a captured frame — and how to reconstruct that structure programmatically.
  • The engineering trade-offs in a dual-mode system design: a unified interface requires careful abstraction so that downstream modules remain mode-agnostic.
  • Graceful degradation as an explicit design goal: a system that falls back cleanly when live capture fails is more useful in practice than one that aborts.
  • Translating raw numerical data into meaningful aggregate statistics using pandas — aggregation, groupby, and time-series resampling.
  • The value of zero-dependency reproducibility: shipping a synthetic dataset generator inside the project means any reviewer can verify the output in under five minutes.
  • Writing typed, testable Python: using dataclasses to represent parsed packets makes the analyzer module straightforward to unit-test in isolation.
