# 即時封包監測與流量統計系統

**Real-time Packet Monitoring and Traffic Statistics System**

---

## Project Overview

This project is a student-driven, research-style implementation of a real-time packet monitoring and traffic statistics system, developed as an introductory hands-on exercise in network and communication engineering. The system is written entirely in Python 3 and is designed to be reproducible across platforms, making it suitable both as a course deliverable and as a foundation for more advanced research.

At its core, the project demonstrates competency in four interconnected domains: real-time packet monitoring via live network sniffing, multi-layer protocol dissection (Ethernet → IPv4 → TCP/UDP/ICMP/DNS/HTTP), flow-level traffic profiling using statistical aggregation, and the translation of raw packet data into structured visualizations and reports. The dual-mode architecture — supporting both offline `.pcap` analysis and live interface capture with graceful permission fallback — reflects a deliberate engineering trade-off between portability and depth. This project is intended as a starting point that can naturally evolve into a more formal research initiative, such as anomaly detection over captured flows, ML-based traffic classification, or integration with open-source network observability stacks like Zeek or Elasticsearch.

在設計上，本系統刻意強調 **real-time packet inspection** 與 **traffic profiling** 兩個面向：前者讓我們理解封包在傳輸瞬間的結構（IP 來源與目的、協定類型、埠號、TTL 等）；後者則讓我們從時間維度觀察流量趨勢，例如每秒封包數（PPS）的起伏、各協定占比的分布、以及尖峰流量的出現時間點。這兩者的結合，正是現代網路監控系統（如 Wireshark、Zeek、ntopng）的核心設計哲學。

本專題的完成不僅是一份課程成果，更是一次系統設計思維的演練：如何將複雜的網路資料拆解為可處理的模組？如何在不同作業系統、不同權限環境下維持程式的可執行性？如何讓分析結果對非技術背景的讀者也具備可讀性？這些問題的思考過程，促使我更深刻地理解了 **networked systems** 與 **communication engineering** 的交界，也讓我對未來在這個領域繼續深化研究充滿期待。

## 🎓 For Admissions Committee

This project was independently designed and implemented as an undergraduate-level technical portfolio piece. The following outlines the competencies it demonstrates across key evaluation dimensions:

### Network Protocol Understanding
- Implements multi-layer **protocol dissection** from raw bytes up through Ethernet, IPv4, TCP, UDP, ICMP, DNS, and basic HTTP detection.
- Demonstrates a working understanding of network stack layering, header field semantics (TTL, port numbers, TCP flags), and the role of protocols at each OSI layer.
- Correctly handles edge cases such as non-IP traffic, malformed layers, and protocol ambiguity (e.g., DNS over UDP).

### System Design & Modularization
- The codebase is organized into six independent modules (`capture`, `parser`, `analyzer`, `dashboard`, `report`, `utils`), each with a single clear responsibility.
- A unified capture interface abstracts the difference between offline pcap reading and live network sniffing, enabling the rest of the system to remain mode-agnostic.
- Graceful degradation is built in: live capture failure does not crash the program but instead falls back to a synthetic demo dataset, illustrating defensive engineering practice.

### Data Analysis & Visualization
- Uses **pandas** to aggregate packet-level data into flow statistics: packet counts, byte totals, packets-per-second (PPS), protocol distribution, and top-N endpoint rankings.
- Generates publication-quality PNG charts (pie chart for protocol distribution, horizontal bar chart for top endpoints) via **Matplotlib**.
- Exports a structured CSV file for downstream analysis and a formatted Markdown report suitable for course portfolios or technical presentations.

### Documentation & Reproducibility
- The project can be fully reproduced on any machine with Python 3.11+ in under five minutes using the commands in the Installation section below.
- A synthetic `demo.pcap` is auto-generated by the system itself (no external dataset required), ensuring zero-dependency reproducibility.
- The codebase includes 18 unit tests (pytest), inline docstrings on all public functions, and this README as a self-contained reference document.

## ✨ Technical Highlights

- **Dual-mode architecture** — supports both offline `.pcap` analysis and live interface capture (`--mode offline` / `--mode live`), with automatic fallback when system privileges are insufficient.
- **Multi-layer protocol dissection** — parses packets from Ethernet frame through IPv4 header down to TCP/UDP/ICMP payload; detects DNS query names and HTTP method verbs from raw bytes.
- **Flow-level statistics** — aggregates per-packet data into time-series metrics including PPS (packets per second), peak traffic timestamps, protocol distribution ratios, and top-N talker rankings.
- **Lightweight network observability** — the entire pipeline runs without a database or external service; all state is held in-memory as a pandas DataFrame, keeping the system portable and self-contained.
- **Rich CLI dashboard** — real-time terminal output using the `rich` library renders colored tables, summary panels, and multi-column layouts without a GUI dependency.
- **Structured data export** — every analysis session can export a CSV (raw packet data), two PNG charts, and a Markdown summary report with a single `--export` flag.
- **Typed, testable design** — packets are parsed into a typed `ParsedPacket` dataclass; the analyzer module is fully unit-tested with 18 pytest cases covering normal operation, edge cases, and empty input.

## 🛠️ Installation

### Prerequisites

- Python 3.11+
- (Linux/macOS live capture) `libpcap`: `sudo apt install libpcap-dev`
- (Windows live capture) [Npcap](https://npcap.com/)

### Setup

```bash
# 1. Clone the repository
git clone https://github.com/TeWei02/Real-time-Packet-Monitoring-and-Traffic-Statistics-System.git
cd Real-time-Packet-Monitoring-and-Traffic-Statistics-System

# 2. Create a virtual environment (recommended)
python -m venv venv
source venv/bin/activate      # Linux/macOS
# venv\Scripts\activate       # Windows

# 3. Install dependencies
pip install -r requirements.txt
```

## 🚀 Usage

### Demo Mode — fastest way to try the system, no pcap file required

```bash
python main.py --demo
```

Automatically generates `sample_data/demo.pcap` with 200 synthetic packets and runs a full analysis.

### Offline Analysis Mode

```bash
# Basic analysis
python main.py --mode offline --pcap sample_data/demo.pcap

# Full analysis with all exports (CSV + charts + Markdown report)
python main.py --mode offline --pcap sample_data/demo.pcap --export

# Analyse only the first 100 packets
python main.py --mode offline --pcap sample_data/demo.pcap --limit 100
```

### Live Capture Mode

```bash
# Linux / macOS (requires root)
sudo python main.py --mode live --iface eth0 --limit 200

# Windows (run as Administrator)
python main.py --mode live --iface "Wi-Fi" --limit 200

# Let Scapy choose the default interface
sudo python main.py --mode live
```

If permission is denied, the system automatically falls back to `demo.pcap` rather than crashing.

### Run Unit Tests

```bash
python -m pytest tests/ -v
```

## ⚠️ Known Limitations

- Live capture requires root / Administrator privileges.
- Scapy on Windows requires Npcap; consider pyshark as an alternative if installation is difficult.
- Only IPv4 traffic is fully parsed; IPv6 packets are labelled `OTHER` and not yet dissected.
- The synthetic `demo.pcap` is randomly generated and does not represent real-world traffic distributions.
